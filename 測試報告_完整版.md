## 🎯 核心測試重點：N+1 優化詳解

### 什麼是 N+1 問題？

假設課程有 10 個作業：
- ❌ **有 N+1 問題**: 
  - 1 次呼叫取得課程資訊
  - 10 次呼叫檢查每個作業是否指定給學生
  - 總共 11 次 API 呼叫

- ✅ **無 N+1 問題**:
  - 1 次呼叫取得課程資訊（包含所有作業的指定資訊）
  - 總共 1 次 API 呼叫

### LMS 系統的優化成果

本次測試顯示：
- 頁面載入時只有 **2 次課程 API 呼叫**（可能是重複請求）
- **0 次 is-designated 查詢** ⭐
- 即使有多個作業，也不會增加 API 呼叫數

### 優化技術

後端 `/api/courses/:id` 端點已經優化：
```javascript
// 一次性查詢所有作業的指定資訊
const assignments = await getAssignmentsWithDesignatedInfo(courseId, userId);

// 返回時包含 isDesignatedForUser 欄位
assignments.forEach(assignment => {
  assignment.isDesignatedForUser = checkIfDesignated(assignment, userId);
});
```

前端直接使用返回的資訊：
```javascript
// 不需要額外 API 呼叫
if (assignment.isDesignatedForUser) {
  // 顯示作業
}
```
