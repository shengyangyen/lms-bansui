# 伴飛計畫 - 激勵系統規格書

> 規劃日期：2026-03-12  
> 狀態：規格確認，待實作

---

## 1. 學員成就系統

### 1.1 等級與經驗值

| 項目 | 說明 |
|------|------|
| 起始等級 | Lv1 |
| 升級條件 | 經驗值滿 100 即升 1 級 |
| 經驗值累積 | 永久累積，升級後不歸零 |

**等級計算公式**（範例）：
- Lv1 → Lv2：0~99 exp
- Lv2 → Lv3：100~199 exp
- Lv3 → Lv4：200~299 exp
- 以此類推：每級需 100 exp

---

## 2. 經驗值來源

### 2.1 完成作業（依評比）

| 評比結果 | 經驗值 | 備註 |
|----------|--------|------|
| 未通過 | 20 | |
| 合格 | 60 | |
| 優異 | 75 | |

**規則**：同一份作業只取最高一次，不重複計算。

### 2.2 參加課程

- 由**管理員手動派發**經驗值
- 統計方式待定（可能：完成課程、出席、參與等）
- 具體規則之後補充

---

## 3. 徽章系統

### 3.1 徽章與經驗值

- 每獲得一個徽章，可得到 **1~3** 經驗值（依徽章類型）
- 徽章由**關鍵事件**觸發

### 3.2 觸發條件範例（用戶之後提供完整表單）

| 徽章類型 | 觸發條件 | 經驗值 |
|----------|----------|--------|
| 範例 A | 每次作業第一次繳交 | 待定 |
| 範例 B | 連續三次通過 | 待定 |
| 範例 C | 連續三次優異 | 待定 |
| 範例 D | 登入達 50 次 | 待定 |
| 範例 E | 生日 | 待定 |
| 範例 F | 第一次發問 | 待定 |

**待用戶提供**：完整徽章清單與觸發條件對照表。

### 3.3 徽章蒐集冊

- 學員可查看已獲得的徽章
- 顯示徽章圖示、名稱、取得時間
- 可區分「已獲得」與「未解鎖」（可選）

---

## 4. 學習歷程記錄

### 4.1 記錄項目

| 類型 | 說明 |
|------|------|
| 參加上課 | 課程參與記錄 |
| 完成作業 | 作業繳交與評比結果 |
| 得到徽章 | 徽章取得記錄 |
| 管理員手動新增 | 可補登、備註 |

### 4.2 功能

- 學員可**隨時回顧**自己的學習旅程
- 時間軸或列表呈現
- 管理員可上傳**照片/圖片**，顯示在旅程中

---

## 5. 首頁設計（個人儀表板）

### 5.1 主要區塊

| 區塊 | 內容 |
|------|------|
| 等級與經驗值 | 目前等級、經驗值進度條 |
| 徽章冊 | 已獲得徽章預覽，可點擊進入完整蒐集冊 |
| 歷史紀錄 | 近期學習歷程摘要 |
| 飛行官公告 | 管理員發布的公告/提醒 |

### 5.2 導航調整

- 登入後進入**個人儀表板**（不再直接進入課程列表）
- 課程列表改為儀表板內的區塊或獨立入口

---

## 6. 金幣與補給站

### 6.1 金幣取得

| 事件 | 金幣 |
|------|------|
| 每次升級 | 待定 |
| 獲得徽章 | 待定 |

### 6.2 補給站

- 金幣可於**補給站**兌換商品
- 商品由管理員上架
- 範例：書、實體證書、實體回憶錄、紀念品等

### 6.3 兌換流程

- 學員選擇商品 → 扣除金幣 → 產生兌換紀錄
- 管理員可查看兌換紀錄，安排寄送/領取

## 7. 資料庫設計（初步）

### 7.1 新增表（待實作時細化）

| 表名 | 用途 |
|------|------|
| `user_levels` | 學員等級、經驗值 |
| `experience_logs` | 經驗值取得紀錄 |
| `badges` | 徽章定義（id, name, trigger_type, exp_reward, coin_reward） |
| `user_badges` | 學員已獲得徽章 |
| `badge_triggers` | 徽章觸發條件設定 |
| `journey_logs` | 學習歷程（類型、描述、時間、附件） |
| `announcements` | 飛行官公告 |
| `coins` / `coin_logs` | 金幣餘額與變動紀錄 |
| `supply_items` | 補給站商品 |
| `coin_exchanges` | 兌換紀錄 |

### 7.2 既有表擴充

- `submissions`：需有評比欄位（未通過/合格/優異）以計算經驗值
- `users`：可能需欄位記錄等級、金幣等（或另表關聯）

## 8. 實作順序建議

### Phase A：基礎架構
1. 資料庫 schema（等級、經驗值、經驗紀錄）
2. 作業評比與經驗值發放邏輯
3. 等級顯示元件

### Phase B：徽章系統
1. 徽章定義表與觸發邏輯
2. 徽章蒐集冊 UI
3. 用戶提供完整徽章表單後，補齊觸發條件

### Phase C：學習歷程
1. 歷程紀錄表與 API
2. 歷程時間軸 UI
3. 管理員上傳圖片功能

### Phase D：首頁改版
1. 個人儀表板頁面
2. 等級、徽章、歷程區塊
3. 飛行官公告區

### Phase E：金幣與補給站
1. 金幣系統（取得、扣除、紀錄）
2. 補給站商品管理（管理員）
3. 學員兌換流程
