## 📋 修復概要

| 優先級 | Bug | 根因 | 修復方式 |

## 🔧 詳細修復

### 1️⃣ 修復 N+1 查詢問題

**問題**:
```javascript
// 舊做法：前端逐個檢查
for (const assignment of data.assignments) {
  const res = await api.get(`/assignments/${assignment.id}/is-designated`);
  if (res.data.isDesignated) designated.add(assignment.id);
}
```
- 10 個作業 = 11 個 API 呼叫（1 個課程 + 10 個作業指定檢查）
- 頁面載入時間線性增長

**修復**:
```javascript
// 新做法：後端一次查詢
const { data } = await api.get(`/courses/${id}`, { headers: { 'x-user-id': userId } });
// data.assignments 已包含 isDesignatedForUser 欄位
```

**後端實裝**:
```javascript
// 一次查詢所有指定
const { data: designations } = await supabase
  .from('assignment_designations')
  .select('assignment_id')
  .in('assignment_id', assignmentIds)
  .eq('student_id', currentUserId);

// 回傳時添加 isDesignatedForUser
assignments: assignments?.map(a => ({
  ...a,
  isDesignatedForUser: designatedMap[a.id] || false
})) || []
```

**效能提升**: 從 N+1 變成 1 個查詢 + 1 個資料處理

### 2️⃣ 修復 Promise.all 故障傳播

**問題**:
```javascript
const [courseRes, assignmentsRes, enrollmentsRes] = await Promise.all([
  api.get(`/courses/${courseId}`),
  api.get(`/courses/${courseId}/assignments`),
  api.get(`/courses/${courseId}/enrollments`)
]);
// 任何一個失敗，整個 fetchData 都失敗
```

**修復**:
```javascript
const results = await Promise.allSettled([/* ... */]);

if (results[0].status === 'fulfilled') {
  setCourse(results[0].value.data.course);
} else {
  console.error('取得課程失敗', results[0].reason);
}
// 其他結果個別處理...
```

**受影響文件**:
- `AssignmentManagement.jsx` L37-55
- `SubmitAssignment.jsx` L25-51

### 3️⃣ 修復缺乏錯誤處理

**位置**: `AssignmentManagement.jsx` L100-110

```javascript
const loadSubmissions = async (assignmentId) => {
  try {
    const target = assignments.find((a) => a.id === assignmentId);
    const endpoint = target?.assignment_type === 'form'
      ? `/assignments/${assignmentId}/form-submissions`
      : `/assignments/${assignmentId}/submissions`;
    const { data } = await api.get(endpoint);
    setSubmissions(data || []);
  } catch (error) {
    console.error('載入提交失敗', error);
    setSubmissions([]);
    throw error;  // 讓調用方知道發生了什麼
  }
};
```

### 4️⃣ 修復 CORS 無限制

**舊代碼**:
```javascript
app.use(cors());  // 允許所有源
```

**新代碼**:
```javascript
const corsOptions = {
  origin: process.env.FRONTEND_URL || 'http://localhost:3000',
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization']
};
app.use(cors(corsOptions));
```

**效果**: 只有來自 FRONTEND_URL 的請求能訪問

---

### 5️⃣ 修復中文檔名編碼問題

**問題**: 下載教材或作業提交時，中文檔名顯示為亂碼

**修復策略**:

1. **儲存原始檔名**:
```javascript
insertData.file_name = req.file.originalname;
```

2. **下載時使用 RFC 5987 UTF-8 編碼**:
```javascript
const downloadFilename = material.file_name || `${material.title}.${ext}`;
res.setHeader('Content-Disposition', 
  `attachment; filename*=UTF-8''${encodeURIComponent(downloadFilename)}`);
```

**適用位置**:
- 教材上傳與下載
- 作業提交與下載
- 批改結果輸出
