# 作業下載異常 — 檢查方式（白話版 + 技術附錄）

---

## 給你的「超簡單版」— 照順序做就好

不用懂程式。你只要**照下面做**，並把**打勾或寫一句話**的結果留給自己或給工程師，就夠我們鎖定問題。

---

### 第 1 步：下載下來的「檔案」長什麼樣？

在批改畫面，再按一次**下載學員作業**，存到桌面後：

- [ ] **A.** 用記事本打開，裡面是像 `{"error":"……"}` 這種**英文大括號** → 代表網站其實回的是**錯誤訊息**，不是真的檔案。  
  **請把引號裡那句中文／英文錯誤整句抄下來。**

- [ ] **B.** 用記事本打開，裡面是**一整片網頁**（看得到 `<html` 或很多標籤）→ 代表下載到的是**網頁**，不是作業檔。

- [ ] **C.** 副檔名是對的（例如 `.docx`、`.jpg`），但 Word／圖片軟體**說壞掉或打不開** → 可能是**內容在傳送時壞了**，或**其實不是真的那種檔案**（只是被取成那個副檔名）。

- [ ] **D.** 按下去就**沒反應**、或瀏覽器說**沒權限／登入過期** → 多半是**登入狀態**問題，可先**登出再登入**試一次，再記下還是不是 D。

**你這一步只要勾 A/B/C/D，若有抄錯誤句子就附上。**

---

### 第 2 步：Google 雲端硬碟裡，同一個檔案好不好？

（你之前說過雲端裡**可以正常開**，若仍如此，這步打勾就好。）

- [ ] 我在雲端硬碟找到**同一個學生、同一次繳交**的檔案，**用電腦直接開是正常的**。

若 **第 1 步是 C（副檔名對但打不開）** 且 **第 2 步有打勾** → 很常代表：**雲端有檔，但網站「轉手下載」那一段壞了**（這就是我們程式要修的地方）。

---

### 第 3 步：在 Supabase 看一眼（像 Excel 一樣的表）

1. 登入 **Supabase** → 左邊選你的專案。  
2. 左欄 **Table Editor** → 點表格 **`submissions`**。  
3. 找到**出問題的那一次繳交**那一列（若不知道哪一列，可先問工程師要「submission 的 id」，或從網址／畫面上抄 UUID）。

看這三個欄位（有就寫「有」、空的寫「沒有」）：

| 欄位名稱 | 你有看到內容嗎？（有／沒有） |
|----------|------------------------------|
| `drive_file_id` | |
| `file_url` | |
| `file_name` | |

**怎麼解讀（給你安心用）：**

- **`drive_file_id` 有內容** → 系統預期是從 **Google 雲端**抓檔給你下載。  
- **`file_url` 有內容、但沒有雲端 id** → 比較像存在**網站主機自己的資料夾**；在 **Render** 這種主機上，檔有時會不見，下載就容易怪。  

你只要填上表，不用懂細節。

---

### 第 4 步（選做）：用「開發者工具」多拍一張照

若你願意多 2 分鐘，可以幫大忙：

1. 在批改頁按 **F12**（或右鍵 → 檢查）。  
2. 上方點 **「網路」或 Network**。  
3. 再按一次**下載**。  
4. 在列表裡找名字裡有 **`download`** 的那一列，點一下。  
5. **截圖**給工程師，或自己看右邊有沒有寫 **404、403、401** 這種數字。

**看不懂沒關係，截圖就很有用。**

---

### 第 5 步（選做）：主機後台有沒有罵人

若網站是架在 **Render**：

1. 登入 Render → 點你的**後端服務**。  
2. 左邊 **Logs**。  
3. 再按一次下載，然後在 Log 搜尋 **`drive`** 或 **`下載`**。  
4. 若有紅字或明顯錯誤，**複製那幾行**貼給工程師。

---

## 你只要交給工程師的「最小包」

寫在紙上或訊息裡也可以：

1. 第 1 步：**A / B / C / D**（若有錯誤句子一併貼上）。  
2. 第 2 步：**雲端檔案能不能正常開**（能／不能）。  
3. 第 3 步：**三個欄位有沒有內容**（小表）。  
4. 有沒有：**Network 截圖**、**Render 錯誤幾行**（選做）。

這樣通常就夠判斷要改「權限」「雲端串接」還是「程式少讀一個欄位」等，而不用你一直試錯。

## 附錄：技術向檢查（工程師用）

> 以下保留給對照程式與進階除錯；一般使用者可略過。

### 下載種類與 Network

| 項目 | 填寫 |
|------|------|
| 學員繳交 / 批改附件 | |
| 實際 URL path | `/api/submissions/.../download` 或 `/api/feedback/.../download` |
| HTTP 狀態 | 200 / 401 / 403 / 404 |
| Content-Type | `application/octet-stream` / `application/json` / `text/html` |

判讀：200 但 body 為 JSON/HTML → 錯誤頁被存成檔案；401/403 → 權限與 Token。

### 資料庫 `submissions`

- `drive_file_id` 有值 → 應走 Drive。  
- 僅 `file_url` → 本機 `uploads`；Render 上易失效。

### 程式稽核（已知可疑點）

`backend/src/assignments.js` — `GET /submissions/:submissionId/download`：

- `select` 若**未含 `drive_file_id`**，則即使 DB 有值，程式也拿不到，**永遠不會走 `downloadFromDrive`**，易誤走 `file_url` 或報錯。  
- 與「Drive 上檔案正常、下載異常」一致，**建議與第 3 步 DB 欄位一併核對後再修**。

### 其他

- `driveService.js`、Render 環境變數 `GOOGLE_DRIVE_*`。  
- `docs/開發歷程.md` 問題 4（HTML）、7（MIME/副檔名）、8（須走下載 API）。

### curl（可選）

```bash
curl -sI -H "Authorization: Bearer TOKEN" "https://HOST/api/submissions/SUBMISSION_ID/download"
```

## 確認問題後再修正

建議至少完成**白話版第 1～3 步**；工程師再依附錄對程式下刀，避免同一原因反覆修改。
