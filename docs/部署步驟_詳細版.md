## 第一步：把程式碼放到 GitHub

### 為什麼要做？

Render 和 Vercel 都是從 GitHub 讀取你的程式碼來部署，所以要先上傳到 GitHub。

### 1-1 建立 GitHub 倉庫

1. 打開瀏覽器，到 [https://github.com/new](https://github.com/new)
2. **Repository name**：輸入一個名字，例如 `lms-bansui`（英文、可加數字或底線）
3. **Description**：可留空
4. 選擇 **Private**（私人）或 **Public**（公開）
5. **不要**勾選「Add a README file」
6. 點綠色按鈕 **Create repository**

### 1-2 複製倉庫網址

建立完成後，會看到一個網址，例如：

```
https://github.com/shengyangyen/lms-bansui.git
```

把這個網址複製起來，等一下會用到。

### 1-3 在電腦上執行指令

1. 按 `Win + R`，輸入 `powershell`，按 Enter 開啟 PowerShell
2. 複製貼上下面這行，按 Enter（會切到專案資料夾）：
  ```
   cd "c:\Users\gn020\Desktop\我的vibe coding工具\LMS"
  ```
3. 複製貼上下面這行，**把 `你的帳號/你的倉庫名` 改成你剛建立的倉庫**，例如 `gn020/lms-bansui`：
  ```
   git remote add origin https://github.com/shengyangyen/lms-bansui.git
  ```
4. 複製貼上：
  ```
   git branch -M main
  ```
5. 複製貼上（會上傳程式碼，可能需要輸入 GitHub 帳密或授權）：
  ```
   git push -u origin main
  ```

如果第 5 步要求登入，請用 GitHub 的帳號密碼，或依照畫面指示用 Token 登入。

## 第二步：Render 部署後端

### 為什麼要做？

後端（API、資料庫連線、寄信）需要 24 小時在網路上跑，Render 會幫你做到這件事。

### 2-1 建立 Web Service

1. 到 [https://dashboard.render.com](https://dashboard.render.com)
2. 點左上角 **New +** → 選 **Web Service**
3. 若還沒連接 GitHub，會要你「Connect GitHub」→ 選你的帳號 → 授權
4. 在倉庫列表找到你剛建立的 `lms-bansui`（或你取的名字），點 **Connect**

### 2-2 填寫設定


| 欄位                 | 填什麼                     | 說明                          |
| ------------------ | ----------------------- | --------------------------- |
| **Name**           | `lms-backend`           | 服務名稱，之後網址會用到                |
| **Region**         | 選離台灣近的（如 Singapore）     | 伺服器位置                       |
| **Branch**         | `main`                  | 預設即可                        |
| **Root Directory** | 留空（或填 `backend`）        | 若留空，Build/Start 需用下方指令      |
| **Runtime**        | `Node`                  | 預設即可                        |
| **Build Command**  | `cd backend && npm install` | 安裝 backend 套件               |
| **Start Command**  | `npm start`               | 根目錄的 start 會執行 backend     |
| **Start Command**  | `npm start`             | 啟動後端                        |


### 2-3 設定環境變數（重要）

往下捲，找到 **Environment** 區塊，點 **Add Environment Variable**，一筆一筆加入：


| Key             | Value（從哪裡找）                              |
| --------------- | ---------------------------------------- |
| `SUPABASE_URL`  | 打開 backend/.env，複製 SUPABASE_URL 那行的值     |
| `SUPABASE_KEY`  | 同上，複製 SUPABASE_KEY                       |
| `JWT_SECRET`    | 同上，複製 JWT_SECRET（若沒有，可自己打一串 32 字以上的英文數字） |
| `FRONTEND_URL`  | 先填 `https://lms.vercel.app`（暫定，之後會改）     |
| `SMTP_HOST`     | 從 .env 複製 SMTP_HOST                      |
| `SMTP_PORT`     | `587`                                    |
| `SMTP_EMAIL`    | 從 .env 複製 SMTP_EMAIL                     |
| `SMTP_PASSWORD` | 從 .env 複製 SMTP_PASSWORD                  |
| `NODE_ENV`      | `production`                             |


**注意**：Value 不要加引號，直接貼上值即可。

### 2-4 建立並等待部署

1. 點最下方藍色按鈕 **Create Web Service**
2. 等待 3～5 分鐘，畫面會顯示「Building...」→「Deploying...」→「Live」
3. 部署成功後，上方會出現網址，例如：`https://lms-backend-xxxx.onrender.com`
4. **把這個網址複製下來**，等一下 Vercel 會用到

## 第三步：Vercel 部署前端

### 為什麼要做？

前端（登入頁、課程頁等）需要讓大家用瀏覽器打開，Vercel 會幫你放到網路上。

### 3-1 建立專案

1. 到 [https://vercel.com](https://vercel.com)，用 GitHub 登入
2. 點 **Add New...** → **Project**
3. 在 **Import Git Repository** 找到你的 `lms-bansui`，點 **Import**

### 3-2 填寫設定


| 欄位                   | 填什麼                      | 說明                         |
| -------------------- | ------------------------ | -------------------------- |
| **Project Name**     | 可保持預設（如 lms-bansui）      | 會變成網址的一部分                  |
| **Root Directory**   | 點 **Edit**，輸入 `frontend` | 告訴 Vercel 前端在 frontend 資料夾 |
| **Framework Preset** | 選 **Vite**               | 若沒自動偵測，手動選                 |
| **Build Command**    | `npm run build`          | 預設通常已填好                    |
| **Output Directory** | `dist`                   | 預設通常已填好                    |


### 3-3 部署

1. 點 **Deploy**
2. 等待 1～2 分鐘
3. 完成後會顯示網址，例如：`https://lms-bansui.vercel.app`
4. **把這個網址複製下來**

## 第四步：把前後端連起來

### 4-1 修改 vercel.json

1. 用 Cursor 或記事本打開專案裡的 `frontend/vercel.json`
2. 找到這行：
  ```json
   "destination": "https://YOUR-RENDER-APP.onrender.com/api/:path*"
  ```
3. 把 `YOUR-RENDER-APP` 換成你 Render 網址中間那段，例如：
  - Render 網址是 `https://lms-backend-abc1.onrender.com`
  - 就改成：`"destination": "https://lms-backend-abc1.onrender.com/api/:path*"`
4. 存檔

### 4-2 修改 Render 的 FRONTEND_URL

1. 回到 Render Dashboard，點你的 `lms-backend` 服務
2. 左側點 **Environment**
3. 找到 `FRONTEND_URL`，點右邊的 **Edit**（鉛筆圖示）
4. 把值改成你的 Vercel 網址，例如：`https://lms-bansui.vercel.app`
5. 儲存

### 4-3 重新部署

**Vercel：**

1. 在 PowerShell 執行：
  ```
   cd "c:\Users\gn020\Desktop\我的vibe coding工具\LMS"
   git add frontend/vercel.json
   git commit -m "Update vercel.json with Render URL"
   git push
  ```
2. Vercel 會自動偵測到 push，重新部署（約 1 分鐘）

**Render：**

1. 在 Render 你的服務頁面，點右上角 **Manual Deploy** → **Deploy latest commit**
2. 等待部署完成

## 常見問題

**Q：Render 顯示「Build failed」？**  
A：檢查 Root Directory 是否正確填 `backend`，以及環境變數是否都有填。

**Q：登入後一直轉圈或 404？**  
A：檢查 vercel.json 的 destination 是否正確指向 Render 網址，且網址結尾要有 `/api/:path`*。

**Q：教材下載失敗？**  
A：Render 的磁碟是暫時的，重啟後上傳的檔案會消失。上線後需要重新上傳教材，或之後做 Google Drive 整合。

**Q：Git push 失敗？**  
A：確認 GitHub 倉庫網址正確，且你有該倉庫的寫入權限。
