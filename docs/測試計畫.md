## 🧪 測試流程

### PHASE 1️⃣：帳號系統 (Accounts)

#### Test 1.1 - 註冊流程
- [ ] 進 http://localhost:3000/register
- [ ] 填寫表單（暱稱、真實姓名、信箱、密碼、確認密碼）
- [ ] 提交 → 出現「請檢查信箱」提示
- [ ] **預期**：帳號在資料庫 `status='pending'`

#### Test 1.2 - Admin 審核帳號
- [ ] Admin 登入 → 進管理面板
- [ ] 點「帳號審核」頁籤
- [ ] 看到新註冊的帳號
- [ ] 點「批准」→ 選擇角色（instructor）
- [ ] **預期**：帳號變 `status='approved'` + `user_role='instructor'`

#### Test 1.3 - 教師登入
- [ ] 用新帳號登入（教師@example.com）
- [ ] **預期**：進入 Dashboard，看到教師專屬的管理選單

#### Test 1.4 - 拒絕帳號
- [ ] 新註冊一個帳號（學生 B）
- [ ] Admin 審核頁籤點「拒絕」
- [ ] 用該帳號嘗試登入
- [ ] **預期**：提示「帳號申請已被拒絕」

#### Test 1.5 - 用戶管理
- [ ] Admin 進「用戶管理」頁籤
- [ ] 看到所有已批准用戶列表
- [ ] 編輯一個用戶的角色（學生A → trainee）
- [ ] 重設該用戶的密碼
- [ ] **預期**：修改成功，用新密碼能登入

---

### PHASE 2️⃣：課程系統 (Courses)

#### Test 2.1 - 建立課程
- [ ] 教師登入 → 管理面板
- [ ] 點「課程管理」
- [ ] 填寫課程名稱、描述
- [ ] 建立
- [ ] **預期**：課程出現在列表中

#### Test 2.2 - 管理學員
- [ ] 在課程卡片點「👥 管理學生」
- [ ] 在「可新增學生」區選 2 個學生（A + B）
- [ ] 點「加入」
- [ ] **預期**：學生出現在「已註冊學生」區

#### Test 2.3 - 學生看得到課程
- [ ] 學生 A 登入
- [ ] 進 Dashboard → 課程列表
- [ ] **預期**：只看得到被 enroll 的課程

#### Test 2.4 - 移除學生
- [ ] 教師回到「👥 管理學生」
- [ ] 點「移除」移除學生 B
- [ ] 學生 B 登入驗證
- [ ] **預期**：學生 B 看不到該課程

---

### PHASE 3️⃣：教材系統 (Materials)

#### Test 3.1 - 上傳檔案教材
- [ ] 教師進課程管理 → 教材管理
- [ ] 上傳一份 PDF / Word 檔案
- [ ] 填寫標題、描述
- [ ] 提交
- [ ] **預期**：教材出現在列表

#### Test 3.2 - 設定教材可見時間
- [ ] 編輯教材 → 設定「可見時間」（未來某個時間）
- [ ] 學生現在看不到
- [ ] 等待時間過期或改為過去時間
- [ ] **預期**：教材出現

#### Test 3.3 - 添加外部連結
- [ ] 新增教材 → 選「🔗 添加連結」
- [ ] 貼 YouTube URL
- [ ] **預期**：學生端自動內嵌播放器

#### Test 3.4 - 下載教材
- [ ] 學生進課程 → 教材頁籤
- [ ] 點「📥 下載檔案」
- [ ] **預期**：正確下載，檔名無亂碼

---

### PHASE 4️⃣：作業系統 - 基本 (Assignments - Basic)

#### Test 4.1 - 建立檔案上傳作業
- [ ] 教師進課程 → 作業管理 → 建立作業
- [ ] 類型：檔案上傳
- [ ] 標題、描述
- [ ] **指定對象**：勾選「學生 A」只 → **不勾學生 B**
- [ ] 建立
- [ ] **預期**：作業出現在列表

#### Test 4.2 - 編輯作業（修改指定）
- [ ] 點作業卡片的「✏️ 編輯」
- [ ] 改成勾選「學生 B」
- [ ] 保存
- [ ] **預期**：作業指定對象改變

#### Test 4.3 - 學生看指定作業
- [ ] 學生 A 登入 → 課程 → 作業頁籤
- [ ] **預期**：看不到該作業（因為指定給了 B）
- [ ] 學生 B 登入
- [ ] **預期**：看得到該作業

#### Test 4.4 - 提交檔案作業
- [ ] 學生 B 點「📤 提交作業」
- [ ] 上傳 PDF / Word
- [ ] 提交
- [ ] **預期**：提交成功，進度顯示「已提交」

#### Test 4.5 - 學生查看提交歷程
- [ ] 學生重新進作業
- [ ] 看到「查看反饋」按鈕
- [ ] 點進去看版本歷程
- [ ] **預期**：能看到提交時間、批改狀態

---

### PHASE 5️⃣：作業系統 - 填空題 (Assignments - Form)

#### Test 5.1 - 建立填空題作業
- [ ] 教師建立作業 → 類型：填空題作答
- [ ] 新增 3 題：
  - 題目 1：「你的名字」
  - 題目 2：「最喜歡的程式語言」
  - 題目 3：「為什麼選這個」
- [ ] **指定對象**：全班（都不勾）
- [ ] 建立
- [ ] **預期**：作業建立成功

#### Test 5.2 - 學生填答
- [ ] 學生 A 進作業 → 「📝 前往作答」
- [ ] 逐題填寫
- [ ] 點「提交」
- [ ] **預期**：提交成功

#### Test 5.3 - 學生重新提交（新版本）
- [ ] 同學生 A 再進作業
- [ ] 修改答案
- [ ] 再次提交
- [ ] **預期**：`version_number` 變成 2

#### Test 5.4 - 匯出答案
- [ ] 教師查看提交 → 點「📄 匯出 PDF」和「📊 匯出 Excel」
- [ ] **預期**：檔案正確下載，內容包含學生答案

---

### PHASE 6️⃣：批改系統 (Grading)

#### Test 6.1 - 教師批改作業
- [ ] 教師進「查看提交」tab
- [ ] 看到學生的提交列表
- [ ] 點某個提交 → 進批改頁
- [ ] 選評分等級（例如「優秀」）
- [ ] 輸入批改意見
- [ ] 上傳批改檔案（可選）
- [ ] 提交批改
- [ ] **預期**：批改記錄保存

#### Test 6.2 - 學生看批改反饋
- [ ] 學生進作業 → 「查看反饋」
- [ ] 看到教師的評分和意見
- [ ] 看到版本歷程
- [ ] **預期**：能清楚看到整個迭代過程

#### Test 6.3 - 批改後重新提交
- [ ] 學生再提交一次作業（改過的版本）
- [ ] 教師再次批改（新評分）
- [ ] **預期**：
  - 有新版本記錄
  - 舊版本 + 舊評分不消失
  - 版本歷程清楚

#### Test 6.4 - 統計頁面
- [ ] 教師進作業統計
- [ ] 看到「應繳交」、「已繳交」、「待批改」等卡片
- [ ] 點某個卡片看詳細列表
- [ ] **預期**：數字正確（只計被指定的學生）

#### Test 7.1 - 課程討論
