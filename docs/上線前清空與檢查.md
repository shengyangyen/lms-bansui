# 上線前清空與檢查

## 一、通知信檢查

若教材上架、作業批改、作業上線時沒有收到通知信：

### 1. 檢查 Render 環境變數

到 Render Dashboard → 你的服務 → **Environment**，確認有這四筆：

| Key | 範例值 |
|-----|--------|
| `SMTP_HOST` | smtp.gmail.com |
| `SMTP_PORT` | 587 |
| `SMTP_EMAIL` | 你的信箱@gmail.com |
| `SMTP_PASSWORD` | Gmail 應用程式密碼（16 字元） |

### 2. Gmail 應用程式密碼

若用 Gmail，**不能用一般密碼**，需建立應用程式密碼：

1. Google 帳號 → 安全性 → 兩步驟驗證（需先開啟）
2. 應用程式密碼 → 選「郵件」與「Windows 電腦」→ 產生
3. 複製產生的 16 字元密碼，貼到 Render 的 `SMTP_PASSWORD`

### 3. 檢查 API

開啟 `https://lms-bansui.onrender.com/api/email-status`：

- `emailEnabled: true` → SMTP 已設定，若仍沒信請看 Render Logs
- `emailEnabled: false` → 請補上上述環境變數並重新部署

### 4. 查看 Render Logs

觸發教材上線或批改後，到 Render → Logs，搜尋：

- `[email] SMTP 已啟用` → 啟動時有看到代表 SMTP 正常
- `[email] SMTP 連線驗證失敗` → SMTP 設定有誤
- `[email] 教材通知寄送失敗` → 寄信時出錯（密碼錯誤等）

## 二、清空資料庫（教材、課程、作業）

1. 前往 Supabase → SQL Editor
2. 複製貼上 `database/clear_all_courses_materials_assignments.sql` 的內容
3. 執行（Run）
4. **注意**：會刪除所有課程、教材、作業、提交、批改，**用戶帳號會保留**

---

## 三、清空 Google Drive 測試檔案

1. 登入有 LMS-System 共用雲端硬碟的 Google 帳號
2. 開啟該共用雲端硬碟
3. 手動刪除測試檔案（如 `lms-test-upload.txt` 及先前上傳的教材、作業）

---

## 四、帳號權限

**可以**手動移除後再重新設定。

- **方式一**：透過 LMS 後台（若有用戶管理頁面）修改角色、移除學員
- **方式二**：在 Supabase Table Editor 直接編輯 `users` 表的 `role` 欄位
- **方式三**：刪除測試帳號後重新註冊

`course_enrollments` 表記錄學員與課程的對應，若要重設選課，可執行：
```sql
DELETE FROM course_enrollments;
```
