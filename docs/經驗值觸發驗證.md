# 經驗值觸發驗證指南

## 資料庫遷移

- **動態通知**：執行 `database/add_activity_notifications.sql` 建立 `activity_notifications` 表
- **訊息聯絡**：執行 `database/add_contact_messages.sql` 建立 `contact_messages` 表

## 目前觸發機制

### ✅ 作業評分（已實作）

**觸發時機**：教師對學員作業進行批改並給予評分時

**流程**：
1. 學員繳交作業（上傳檔案 或 填空題）
2. 教師進入作業批改頁面，選擇評分等級並送出
3. 系統呼叫 `POST /api/submissions/:submissionId/feedback`，body 含 `{ grade: '建議需調整'|'合格'|'優秀' }`
4. 後端在寫入 feedback 後呼叫 `awardAssignmentExp(studentId, assignmentId, grade)`
5. 經驗值寫入 `experience_logs` 與 `user_levels`

**評分對應**：
| 評分等級     | 經驗值 |
|-------------|--------|
| 建議需調整   | 20     |
| 合格        | 60     |
| 優秀        | 75     |

**驗證步驟**：
1. 以學員身分登入，繳交一筆作業
2. 以教師身分登入，進入該作業的批改頁面
3. 選擇評分（合格或優秀）並送出
4. 學員重新整理 Dashboard，確認經驗值與等級有增加
5. 在管理面板「經驗值管理」確認該學員總經驗值已更新
6. Dashboard「最近動態」應顯示「完成作業《XXX》獲得 N 經驗值」

---

### ❌ 上課（未實作）

目前沒有「上課」或「完成課程」的經驗值觸發。若需要，可新增：
- 學員完成某課程單元時發放經驗值
- 或學員首次進入課程時發放

---

### 繳交作業

**繳交本身不發放經驗值**，只有**被評分後**才會發放。同一作業若多次批改，只取最高評分對應的經驗值。
