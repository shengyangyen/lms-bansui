# LMS 部署步驟（Vercel + Render）

## 前置：GitHub 倉庫

1. 在 GitHub 建立新倉庫（例如 `lms-bansui`），**不要**勾選 README
2. 本地執行：

```powershell
cd "c:\Users\gn020\Desktop\我的vibe coding工具\LMS"
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的帳號/你的倉庫名.git
git push -u origin main
```

---

## 步驟 1：Render 部署後端

1. 登入 [Render](https://render.com)
2. **New** → **Web Service**
3. 連接你的 GitHub 倉庫
4. 設定：
   - **Name**：`lms-backend`（或自訂）
   - **Root Directory**：`backend`
   - **Runtime**：Node
   - **Build Command**：`npm install`
   - **Start Command**：`npm start`
5. **Environment** 新增變數（從 backend/.env 複製，**勿上傳 .env 檔**）：

   | Key | Value |
   |-----|-------|
   | SUPABASE_URL | 你的 Supabase URL |
   | SUPABASE_KEY | 你的 Supabase key |
   | JWT_SECRET | 至少 32 字元隨機字串 |
   | FRONTEND_URL | `https://你的vercel網址.vercel.app`（先填暫定，Vercel 部署後再改） |
   | SMTP_HOST | 你的 SMTP 主機 |
   | SMTP_PORT | 587 |
   | SMTP_EMAIL | 寄件信箱 |
   | SMTP_PASSWORD | SMTP 密碼 |
   | NODE_ENV | production |

6. 點 **Create Web Service**，等待部署完成
7. 記下網址，例如：`https://lms-backend-xxxx.onrender.com`

---

## 步驟 2：Vercel 部署前端

1. 登入 [Vercel](https://vercel.com)
2. **Add New** → **Project**，匯入你的 GitHub 倉庫
3. 設定：
   - **Root Directory**：`frontend`（點 Edit 設定）
   - **Framework Preset**：Vite
   - **Build Command**：`npm run build`
   - **Output Directory**：`dist`
4. **Deploy**，等待完成
5. 記下網址，例如：`https://lms-xxx.vercel.app`

---

## 步驟 3：更新 vercel.json 與 FRONTEND_URL

1. 編輯 `frontend/vercel.json`，將 `YOUR-RENDER-APP` 改為你的 Render 服務名稱：
   - 若 Render 網址是 `https://lms-backend-xxxx.onrender.com`
   - 則 destination 改為：`https://lms-backend-xxxx.onrender.com/api/:path*`

2. 到 Render Dashboard → 你的服務 → **Environment** → 編輯 `FRONTEND_URL` 為你的 Vercel 網址

3. 重新部署：
   - Vercel：push 更新後的 vercel.json 到 GitHub，或 Vercel 會自動重新部署
   - Render：手動 **Manual Deploy** → **Deploy latest commit**（讓 FRONTEND_URL 生效）
