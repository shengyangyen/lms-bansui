# 環境變數與部署檢查

## 一、Backend 環境變數

| 變數 | 必填 | 說明 |
|------|------|------|
| `SUPABASE_URL` | ✅ | Supabase Project URL |
| `SUPABASE_KEY` | ✅ | Supabase anon key（或 service role，視權限需求） |
| `JWT_SECRET` | ✅ | 至少 32 字元，用於簽署 JWT |
| `FRONTEND_URL` | ✅ | 前端網址，用於 CORS、Email 連結（如 `https://your-app.vercel.app`） |
| `SMTP_HOST` | ✅ | SMTP 主機 |
| `SMTP_PORT` | ✅ | 通常 587 |
| `SMTP_EMAIL` | ✅ | 寄件者信箱 |
| `SMTP_PASSWORD` | ✅ | SMTP 密碼 |
| `NODE_ENV` | 選填 | `production` / `development` |
| `PORT` | 選填 | 預設 3001 |
| `LEADERBOARD_ALWAYS_VISIBLE` | 選填 | `true` 可強制開放排行榜（除錯用） |

**範本**：`backend/.env.example`，複製為 `.env` 填入實際值。

---

## 二、部署架構（Vercel + Render）

- **前端**：Vercel（靜態 + SPA）
- **後端**：Render（Node.js Web Service）
- **資料庫**：Supabase（已用）

### 部署前檢查

1. **Backend**
   - [ ] Render 建立 Web Service，連到 `backend/` 或 repo root
   - [ ] Start command：`cd backend && npm install && npm start`
   - [ ] 在 Render 設定所有環境變數（勿用 .env 上傳）
   - [ ] `FRONTEND_URL` 設為 Vercel 部署網址

2. **Frontend**
   - [ ] Vercel 部署 `frontend/`，build command：`npm run build`
   - [ ] 需讓 `/api` 轉發到 Render 後端，否則前端會 404

### API 轉發方式

**方案 A：Vercel Rewrites（推薦）**

已建立 `frontend/vercel.json`，部署前將 `YOUR-RENDER-APP` 改為你的 Render 服務名稱。

**方案 B：前端改用絕對 API URL**

在 `frontend/src/api.js` 與 `fetchWithAuth` 中，production 時使用 `import.meta.env.VITE_API_URL`，並在 Vercel 設定 `VITE_API_URL=https://your-backend.onrender.com`。
