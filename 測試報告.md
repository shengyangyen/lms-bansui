- **建議修復**: 
  - 建立標準測試帳號資料

### 🟢 低優先級問題 (20項)

#### 問題分類 1: Token 驗證問題 (13項)
所有需要認證的操作都因為沒有有效 token 而失敗:
- 教師建立課程
- 查詢註冊學生
- 添加/設定教材
- 建立/查看作業
- 提交作業
- 查看批改
- 特殊字符處理測試

**根本原因**: 由於測試帳號登入失敗,無法取得有效的 JWT token

---

#### 問題分類 2: API 路由不存在 (5項)
- **問題描述**: 多個 API 端點返回 404 錯誤
- **受影響的端點**:
  - `GET /api/courses/:courseId/assignments/student` (學生查看作業列表)
  - `POST /api/assignments` (建立作業)
  - `POST /api/assignments/:id/submit-form` (提交填空題)
  - `GET /api/assignments/:id/statistics` (查看作業統計)
  - `GET /api/assignments/:id/submissions/student/:studentId` (查看學生提交歷程)
- **對用戶的影響**: 
  - 作業系統功能無法正常使用
  - 學生無法查看作業、提交作業
  - 教師無法查看統計和學生提交
- **優先級**: **低** (但實際應該是高,只是因為被登入問題掩蓋)
- **建議修復**: 
  - 檢查 `assignments.js` 路由是否正確註冊
  - 確認路由路徑是否與前端期望一致

---

#### 問題 #3: 錯誤 Token 驗證不嚴格
- **測試項目**: 邊界情況測試
- **問題描述**: 系統接受無效的 token (測試發現系統沒有正確拒絕)
- **重現步驟**: 
  1. 使用明顯無效的 token: `Bearer invalid_token_12345`
  2. 發送請求到需要認證的端點
  3. 預期: 返回 403 錯誤
  4. 實際: 可能返回其他狀態碼
- **對用戶的影響**: 可能存在安全漏洞
- **優先級**: **低** (需要進一步驗證)
- **建議修復**: 確認 JWT 驗證中間件是否正確處理無效 token

---

#### 問題 #4: UUID 參數驗證問題
- **測試項目**: 課程/教材查詢
- **問題描述**: `invalid input syntax for type uuid: "null"`
- **重現步驟**: 
  1. courseId 為 null 時查詢課程
  2. PostgreSQL 無法解析 null 為 UUID
- **對用戶的影響**: 
  - API 應該返回 400 錯誤而不是 500 錯誤
  - 用戶體驗不佳
- **優先級**: **低**
- **建議修復**: 在路由層加入參數驗證,在查詢前檢查 UUID 格式

---

#### 問題 #5: N+1 查詢問題
- **測試項目**: 效能測試 - 查詢課程詳情
- **問題描述**: 請求失敗 (400 錯誤)
- **對用戶的影響**: 
  - 無法評估是否存在 N+1 查詢問題
  - 可能影響大量資料時的效能
- **優先級**: **低**
- **建議修復**: 修復課程查詢 API 後重新測試
