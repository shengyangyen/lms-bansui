## ❌ 測試失敗原因分析

### 根本原因 1: 測試帳號不存在或密碼錯誤 (高優先級)

#### 問題描述
測試腳本使用的帳號無法登入:
- `admin@example.com` → 使用者不存在
- `test@example.com` → 密碼錯誤
- `st@example.com` → 密碼錯誤

#### 影響範圍
**所有需要認證的測試都無法執行** (25/32 測試受影響)

#### 受影響功能
- 課程建立與管理
- 教材上傳與設定
- 作業建立與提交
- 批改系統
- 版本管理

#### 解決方案
**選項 A**: 在資料庫中建立標準測試帳號
```sql
-- 建議密碼都設為 'test123'
INSERT INTO users (email, password_hash, display_name, real_name, status, user_role)
VALUES 
  ('admin@test.com', '<bcrypt_hash>', 'Admin', 'Admin User', 'approved', 'admin'),
  ('teacher@test.com', '<bcrypt_hash>', 'Teacher', 'Teacher User', 'approved', 'instructor'),
  ('student1@test.com', '<bcrypt_hash>', 'Student A', 'Student A', 'approved', 'student'),
  ('student2@test.com', '<bcrypt_hash>', 'Student B', 'Student B', 'approved', 'student');
```

**選項 B**: 修改測試腳本使用正確的帳號密碼

**選項 C**: 在測試腳本開始時自動註冊並批准測試帳號

---

### 根本原因 2: API 路由不匹配 (中優先級)

#### 問題描述
測試腳本使用的 API 路由與實際後端路由不一致

#### 路由對照表

| 測試腳本使用的路由 | 實際後端路由 | 狀態 |
|------------------|-------------|------|
| `POST /api/assignments` | `POST /api/courses/:courseId/assignments` | ❌ 不匹配 |
| `GET /api/courses/:courseId/assignments/student` | `GET /api/courses/:courseId/assignments` | ❌ 不匹配 |
| `POST /api/assignments/:id/submit-form` | `POST /api/assignments/:id/form-submit` | ❌ 不匹配 |
| `GET /api/assignments/:id/submissions/student/:studentId` | `GET /api/assignments/:assignmentId/students/:studentId/submissions-history` | ❌ 不匹配 |

#### 影響範圍
導致 5 個測試返回 404 錯誤

#### 解決方案
更新測試腳本以使用正確的 API 路由
