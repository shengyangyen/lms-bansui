## 🆘 常見問題排查

### Q1: 頁面看起來沒有變化（仍是藍色）

**可能原因**:
1. 快取沒有清除
2. Tailwind 沒有重新編譯
3. 開發服務器沒有重啟

**解決方案**:
```bash
# 終端 2 (前端)
# 按 Ctrl+C 停止服務
# 然後重新運行
npm run dev

# 瀏覽器：Ctrl+Shift+Delete 清除快取
# 或用 F12 > Network 禁用快取 > 刷新
```

### Q2: 看到顏色不對劲（例如 #40d7ff）

**可能原因**:
- Tailwind CSS 沒有正確應用全局轉換
- 某個舊的配置仍在運行

**解決方案**:
1. 檢查 `frontend/src/index.css` 是否有內容
2. 檢查 `frontend/tailwind.config.js` 是否正確
3. 檢查 `frontend/src/main.jsx` 是否引入了 `index.css`

### Q3: 字型看起來不對

**可能原因**:
- 系統中沒有安裝相應字型
- 字型載入失敗

**解決方案**:
- 字型會自動 fallback 到系統相似字型
- 檢查 `tailwind.config.js` 的 fontFamily 配置
- 確保 CSS 中有正確的 `font-family` 聲明

### Q4: 按鈕看起來還是實心的（不是外框）

**可能原因**:
- 某個頁面還沒有升級
- 舊的 CSS 類名仍在使用

**解決方案**:
- 檢查該頁面是否在「完整升級」或「部分升級」清單中
- 如果在「待升級」清單中，這是正常的 (色彩已轉換，樣式待升級)
- 參考已升級頁面的代碼進行手動升級
